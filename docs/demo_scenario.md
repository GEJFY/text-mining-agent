## デモシナリオ概要

本デモでは、NexusText AI v7.0 の主要機能を3つのビジネスシナリオを通じて紹介します。各シナリオは独立しており、単独で実施することも、3つ連続で実施することも可能です。

| # | シナリオ名 | 使用データ | 所要時間 | 主な紹介機能 |
|---|----------|----------|---------|------------|
| 1 | **顧客の声(VOC)分析** | `customer_feedback.csv`（100行） | 約20分 | K-Means、Business感情分析、共起ネットワーク、AIエージェント、レポート生成 |
| 2 | **サポート改善分析** | `support_tickets.csv`（40行） | 約15分 | HDBSCAN、Risk感情分析、共起ネットワーク |
| 3 | **組織エンゲージメント分析** | `employee_survey.csv`（50行） | 約15分 | GMM、カスタム感情軸、共起ネットワーク |

### デモデータの概要

**1. customer_feedback.csv（100行）**
- カラム: `id`, `text`, `date`, `category`, `source`
- 期間: 2025年1月〜9月
- 内容: 製品品質、サポート対応、料金・契約、セキュリティ、配送・物流、ドキュメント、システム障害、サービスに関する顧客フィードバック
- ソース: アプリレビュー、問い合わせフォーム、メールアンケート、SNS、電話後アンケート
- 特徴: ポジティブ（製品の高品質、サポートの丁寧さ）とネガティブ（遅延、料金不透明、障害）が混在

**2. support_tickets.csv（40行）**
- カラム: `ticket_id`, `description`, `created_date`, `priority`, `department`, `status`
- 期間: 2025年1月〜4月
- 内容: テクニカルサポート、セキュリティ、経理・請求、アカウント管理、ドキュメントに関するチケット
- 優先度: 緊急/高/中/低の4段階
- ステータス: 解決済み/対応中
- 特徴: 緊急チケット（APIキー漏洩、テスト環境混入、データ漏洩）が含まれる

**3. employee_survey.csv（50行）**
- カラム: `response_id`, `comment`, `survey_date`, `department`, `role`
- 期間: 2025年3月実施
- 内容: 10部署以上からの回答。働き方、評価制度、コミュニケーション、IT環境、福利厚生に関する意見
- 役職: 一般社員、マネージャー、リードエンジニア、シニアエンジニア、アナリスト、研究員など
- 特徴: 満足（リモートワーク、制度改善）と不満（残業、評価不透明、IT環境）が混在

## 事前準備

### 1. アプリケーション起動確認

デモ開始前に、以下の手順でアプリケーションが正常に稼働していることを確認してください。

#### 手順1-1: Docker環境の起動

プロジェクトルートディレクトリで以下のコマンドを実行します。

```bash
docker compose up -d
```

#### 手順1-2: バックエンドAPIの確認

ブラウザまたはcurlで以下のURLにアクセスし、正常なレスポンスが返ることを確認します。

```
http://localhost:8000/api/health
```

期待されるレスポンス:
```json
{"status": "healthy"}
```

#### 手順1-3: フロントエンドの確認

ブラウザで以下のURLにアクセスし、ダッシュボード画面が表示されることを確認します。

```
http://localhost:5173
```

- 画面左側にサイドバーナビゲーションが表示される
- サイドバー上部に「NexusText AI」「v7.0」のロゴが表示される
- メインエリアにダッシュボードのKPIカードとチャートが表示される

#### 手順1-4: デモデータの配置確認

以下の3ファイルが `demo/data/` ディレクトリに存在することを確認します。

```
demo/data/customer_feedback.csv    （100行、約15KB）
demo/data/support_tickets.csv      （40行、約8KB）
demo/data/employee_survey.csv      （50行、約10KB）
```

### 2. ブラウザのセットアップ

## シナリオ1: 顧客の声(VOC)分析 (約20分)

### シナリオの目的

「100件の顧客フィードバックから、顧客満足度の傾向、不満の根本原因、改善優先度を明らかにする」というストーリーでデモを進めます。VOC（Voice of Customer）分析は、テキストマイニングの最も一般的なユースケースであり、聴衆の理解を得やすいシナリオです。

### ステップ1-1: ダッシュボード確認 (2分)

#### 操作手順

1. ブラウザで `http://localhost:5173` にアクセスします
2. 自動的にダッシュボードページ（`/dashboard`）が表示されます

#### 画面の説明ポイント

**KPIカードエリア（画面上部）**

画面上部に4つのKPIカードが横並びで表示されます。それぞれのカードの説明を行います。

| カード名 | アイコン | 説明 |
|---------|---------|------|
| 「登録データセット」 | データベースアイコン（青色） | 現在システムに登録されているデータセットの総数 |
| 「クラスタ分析完了」 | レイヤーアイコン（紫色） | 完了したクラスタリング分析の回数 |
| 「センチメント分析済」 | ハートアイコン（ローズ色） | 感情分析が完了したテキストレコード数 |
| 「生成インサイト」 | 上昇トレンドアイコン（エメラルド色） | AIエージェントが生成したインサイトの総数 |

> **トークポイント**: 「これがNexusText AIのダッシュボードです。上部のKPIカードで、プラットフォーム全体の分析状況をひと目で把握できます。」

**チャートエリア（中段）**

左右に2つのチャートが表示されます。

- **左: センチメントトレンド** - 折れ線グラフで、ポジティブ（緑）、ネガティブ（赤）、ニュートラル（灰色）の月次推移を表示
- **右: クラスタ分布** - ドーナツ型のパイチャートで、各クラスタの件数比率を色分けで表示。右側に凡例（製品品質、カスタマーサポート、価格・コスト、配送・物流、その他）

> **トークポイント**: 「センチメントの推移からトレンドを把握し、クラスタ分布から主要トピックの構成比を見ることができます。」

**下段エリア**

- **左: 月別処理件数** - 棒グラフで月ごとの分析処理件数を表示
- **右: 最近のアクティビティ** - チェックマーク（緑）、時計マーク（黄色アニメーション）でアクティビティ履歴を表示

> **トークポイント**: 「最近のアクティビティからは、チームメンバーの分析作業状況をリアルタイムで確認できます。では、実際にデータをインポートして分析してみましょう。」

**サイドバーナビゲーションの説明**

サイドバーには以下のメニューが表示されています。デモの流れを簡単に説明します。

| メニュー名 | アイコン | 用途 |
|----------|---------|------|
| ダッシュボード | 格子アイコン | 全体サマリー・KPI |
| データインポート | アップロードアイコン | CSV/Excelの取り込み |
| クラスタ分析 | レイヤーアイコン | テキストクラスタリング |
| センチメント分析 | ハートアイコン | 感情・意図分類 |
| 共起ネットワーク | ネットワークアイコン | 単語間の関連性分析 |
| AIエージェント | ロボットアイコン | 自律型分析 |
| レポート生成 | ドキュメントアイコン | PPTX/PDF出力 |

### ステップ1-2: データインポート (3分)

#### 操作手順

1. サイドバーの **「データインポート」** をクリックします
2. 画面上部にステップインジケーターが表示されます:
   `[1 ファイル選択] → [2 カラムマッピング] → [3 プレビュー・確認]`
3. 現在「1 ファイル選択」がアクティブ（紫色背景）になっています

#### ステップ1-2-1: ファイルのアップロード

1. 画面中央に大きなドラッグ&ドロップエリアが表示されています
   - アップロードアイコン（紫色の円形背景）
   - 「ファイルをドラッグ&ドロップ」というテキスト
   - 「またはクリックしてファイルを選択」というリンクテキスト
   - 対応形式: `CSV` `Excel (.xlsx)` `JSON` `TSV`

2. **操作**: ドラッグ&ドロップエリアをクリックします

3. ファイル選択ダイアログが開きます。以下のファイルを選択します:
   ```
   demo/data/customer_feedback.csv
   ```

4. ファイル読み込み中はスピナーアイコンと「ファイルを読み込んでいます...」が表示されます

5. 約1〜2秒後、自動的にステップ2「カラムマッピング」画面に遷移します

> **トークポイント**: 「CSV、Excel、JSONなど様々な形式に対応しています。ドラッグ&ドロップで簡単にデータを取り込めます。」

#### ステップ1-2-2: カラムマッピング

画面上部にアップロード済みファイル情報が表示されます:
- ファイルアイコン（緑色背景のスプレッドシートアイコン）
- ファイル名: `customer_feedback.csv`
- ファイルサイズ、検出カラム数

**「カラムマッピング設定」** セクションで、各カラムの用途を指定します。

> 補足: 「テキスト本文」は必須です。最低1つのカラムを「テキスト本文」に割り当てる必要があります。

以下の通りマッピングを設定します:

| CSVカラム名 | マッピング先（ドロップダウンで選択） | 説明 |
|-----------|------------------------|------|
| `id` | **ID** | レコード識別子 |
| `text` | **テキスト本文** | 分析対象のフリーテキスト（必須） |
| `date` | **日付** | 時系列分析に使用 |
| `category` | **カテゴリ** | フィルタリング・属性分析に使用 |
| `source` | **ソース** | データ取得元の情報 |

**操作手順（各行ごと）**:
1. 各行の右側にあるドロップダウンメニューをクリックします
2. 選択肢が表示されます: テキスト本文 / 日付 / カテゴリ / 著者 / ソース / ID / 無視する
3. 上記の表の通りに選択します
4. 「テキスト本文」を選択した行には緑色のチェックマークが表示されます
5. その他のマッピング済み行には青色のチェックマークが表示されます
6. 「無視する」を選択した行には黄色の警告アイコンが表示されます

> **トークポイント**: 「カラムの用途をここで指定します。テキスト本文は必須で、これが分析対象になります。日付を指定すると時系列分析が可能になり、カテゴリやソースを指定するとクロス集計ができます。」

#### ステップ1-2-3: プレビューと確認

1. 右下の **「プレビュー」** ボタン（目のアイコン付き、紫色）をクリックします

2. **データプレビュー** セクションが表示されます:
   - 先頭5行のデータがテーブル形式で表示されます
   - 各カラムヘッダーの下に、マッピングされた役割がバッジ（緑色）で表示されます
   - 例: `text` カラムの下に「テキスト本文」バッジ

3. **インポート設定確認** セクションに以下が表示されます:
   - ファイル名: `customer_feedback.csv`
   - テキストカラム: `text`
   - 使用カラム数: `5 / 5`（全カラム使用の場合）

4. 設定に問題がないことを確認し、**「インポート実行」** ボタン（アップロードアイコン付き、紫色）をクリックします

5. インポートが完了すると、データセットIDが発行されます

> **トークポイント**: 「プレビュー画面でデータとマッピングを最終確認できます。100件のフィードバックデータが正しくインポートされました。では、このデータをクラスタリングしてみましょう。」

### ステップ1-3: クラスタリング分析 (4分)

#### 操作手順

1. サイドバーの **「クラスタ分析」** をクリックします
2. 左パネルにパラメータ設定、右パネルに結果表示エリアの2カラムレイアウトが表示されます

#### ステップ1-3-1: アルゴリズム選択

左パネルの **「アルゴリズム設定」**（歯車アイコン付き）セクションで、以下の3つのアルゴリズムからK-Meansを選択します。

| アルゴリズム | ラジオボタン | 説明テキスト | 選択 |
|-----------|----------|-----------|------|
| **K-Means** | ○ | 「クラスタ数を指定する標準的な手法」 | **選択する** |
| HDBSCAN | ○ | 「密度ベースのクラスタリング（クラスタ数自動決定）」 | - |
| GMM | ○ | 「ガウス混合モデルによる確率的クラスタリング」 | - |

K-Meansのラジオボタンをクリックすると、その行が紫色の枠で囲まれ選択状態になります。

> **トークポイント**: 「3種類のクラスタリングアルゴリズムを選べます。K-Meansはクラスタ数を事前に指定する最も一般的な手法です。HDBSCANは密度ベースでクラスタ数が自動決定され、GMMは確率的にクラスタを割り当てます。今回はVOC分析なので、想定される5つのカテゴリに合わせてK-Meansを使います。」

#### ステップ1-3-2: パラメータ設定

**「パラメータ調整」** セクションで以下を設定します。

**クラスタ数スライダー**（K-Means / GMM選択時のみ表示）:
- ラベル: 「クラスタ数」
- 範囲: 2〜20
- **設定値: `5`** にスライダーを動かします
- 現在の値が右側に数字で表示されます

**Epsilon (距離閾値) スライダー**:
- ラベル: 「Epsilon (距離閾値)」
- 範囲: 0.00〜2.00
- **設定値: `0.50`** のまま（デフォルト値）
- 外れ値の検出感度を調整するパラメータです

> **トークポイント**: 「クラスタ数を5に設定します。顧客フィードバックには製品品質、サポート対応、料金、セキュリティ、配送など5つ程度のトピックが含まれると予想されるためです。Epsilonは外れ値検出の閾値で、デフォルトのままで十分です。」

#### ステップ1-3-3: 分析実行

1. **「分析実行」** ボタン（再生アイコン付き、紫色）をクリックします
2. ボタンが「実行中...」に変わり、スピナーアイコンが回転します
3. 約2秒後、右パネルに結果が表示されます

> 注意: ボタンの右隣にリセットアイコン（丸い矢印）があります。パラメータをデフォルトに戻したい場合に使用します。

#### ステップ1-3-4: UMAP散布図の確認

右パネル上部に **「UMAP クラスタ散布図」** が表示されます。

**散布図の説明ポイント**:
- 横軸: UMAP-1（次元削減後の第1成分）
- 縦軸: UMAP-2（次元削減後の第2成分）
- 各点はテキストレコードを表し、意味的に近いテキストが近くにプロットされます
- 点の色はクラスタごとに異なります（インディゴ、バイオレット、ピンク、アンバー、エメラルドなど）
- 灰色の薄い点は外れ値（どのクラスタにも属さないテキスト）

**操作**: 各点にマウスをホバーすると、ツールチップでクラスタ名とテキストの一部が表示されます。

> **トークポイント**: 「これがUMAP散布図です。UMAPは高次元のテキスト埋め込みベクトルを2次元に圧縮して可視化する手法です。似た内容のテキストが空間的に近くに配置されます。色がはっきり分かれているので、5つのトピックがうまく分離できていることがわかります。」

#### ステップ1-3-5: クラスタラベルの確認

散布図の下に、クラスタ情報カードが3列のグリッドで表示されます。各カードには以下の情報が含まれます:

| 要素 | 表示内容の例 |
|------|-----------|
| カラードット + クラスタ名 | 「製品品質」（LLMが自動命名） |
| 件数バッジ | 「42件」 |
| キーワードタグ | 「品質」「耐久性」「デザイン」「素材」 |
| コヒーレンススコアバー | 進捗バー + パーセンテージ（例: 82%） |

**操作**: クラスタカードをクリックすると、散布図でそのクラスタの点だけがハイライトされます。もう一度クリックすると全クラスタ表示に戻ります。

> **トークポイント**: 「各クラスタにはAIが自動で名前とキーワードを付けます。コヒーレンススコアはクラスタの一貫性を示す指標で、高いほどそのクラスタ内のテキストが似通っていることを意味します。」

**VOCデータで期待されるクラスタ例**:
- 製品品質（最大クラスタ。UIの使いやすさ、分析機能の精度、バッテリー消費、バグなど多様）
- サポート対応（田中さんの好対応、待ち時間の不満、定型文回答への不満など）
- 料金・契約（複雑な料金プラン、自動課金、解約手続きの不透明さなど）
- セキュリティ（暗号化情報の不足、アクセス権限、SOC2認証への信頼など）
- 配送・システム障害（配送遅延、サイトダウン、メンテナンス通知の遅さなど）

#### ステップ1-3-6: 外れ値リストの確認

クラスタカードの下に **「外れ値一覧（N件）」**（黄色い警告アイコン付き）セクションが表示されます。

- 各行にはテキストID、テキストの一部、UMAP座標が表示されます
- スクロール可能な領域に最大20件の外れ値が表示されます

> **トークポイント**: 「外れ値は、どのクラスタにもうまく当てはまらないテキストです。新たなトピックの兆候や、例外的なフィードバックを発見する手がかりになります。」

### ステップ1-4: 感情分析 (4分)

#### 操作手順

1. サイドバーの **「センチメント分析」** をクリックします

#### ステップ1-4-1: センチメント軸の設定

左パネルの **「センチメント軸設定」** セクションで、Business（ビジネス）モードの軸を設定します。

**デフォルト軸の設定**:

| フィールド | 入力値 |
|----------|--------|
| ポジティブ側ラベル | `満足` |
| ネガティブ側ラベル | `不満` |

1. 「ポジティブ側ラベル」テキストフィールドのデフォルト値「ポジティブ」を消去し、**`満足`** と入力します
2. 「ネガティブ側ラベル」テキストフィールドのデフォルト値「ネガティブ」を消去し、**`不満`** と入力します

> **トークポイント**: 「NexusText AIでは、感情軸を自由にカスタマイズできます。一般的なPositive/Negativeではなく、ビジネスに合わせた軸を設定できます。今回は顧客フィードバックの分析なので、満足・不満に加えて、要望・質問・その他という5分類を使います。」

**カスタム軸の追加**:

Businessモードで分類される5軸（満足/不満/要望/質問/その他）を設定するために、**「カスタム軸を追加」** ボタン（プラスアイコン付き、紫色テキスト）をクリックしてカスタム軸を追加します。

1. **「カスタム軸を追加」** をクリック（1回目）:
   - 軸名: **`要望`** と入力
   - ポジティブ側: **`改善希望`** と入力
   - ネガティブ側: **`現状維持`** と入力

2. **「カスタム軸を追加」** をクリック（2回目）:
   - 軸名: **`質問`** と入力
   - ポジティブ側: **`情報を求めている`** と入力
   - ネガティブ側: **`情報は不要`** と入力

3. **「カスタム軸を追加」** をクリック（3回目）:
   - 軸名: **`その他`** と入力
   - ポジティブ側: **`上記に該当しない`** と入力
   - ネガティブ側: **`-`** と入力

各カスタム軸は灰色の背景ボックスに表示され、右上の x ボタンで削除できます。

> **補足**: バックエンドの `SentimentMode.BUSINESS` プリセットでは、満足/不満/要望/質問/その他の5軸が定義済みです。APIレベルでは `mode: "business"` を指定するだけで自動的に適用されます。ここではUI上でカスタム設定を体験していただく目的でカスタム軸として追加しています。

**「分析設定」** セクションの3つのフィールドに以下を入力します。

|----------|--------|------|

| 2 | 2 | 仮説 | 検証可能な問いを生成 | 紫 |

### AIエージェントの説明

> 「AIエージェントは、人間のデータアナリストの思考プロセスをAIが再現します。まずデータを観察し、仮説を立て、その仮説を検証し、最終的にインサイトとしてまとめます。」

> 「重要なのはHITL（Human-in-the-Loop）の仕組みです。AIが立てた仮説を人間が確認・承認する段階があるので、分析の方向性をコントロールできます。AIの暴走を防ぎつつ、人間の判断力とAIの処理能力を組み合わせた最適な分析が可能です。」

> 「Groundingスコアは、AIの主張がどれだけ元データに裏付けられているかを示す指標です。AIの作り話（ハルシネーション）を検出する仕組みとして機能しています。」

## 想定Q&A

### 技術面の質問

**Q1: 日本語以外の言語にも対応していますか？**

> A: はい、NexusText AIは多言語対応プラットフォームです。日本語、英語、中国語、韓国語などの主要言語に対応しています。LLMの多言語能力を活用しているため、言語を指定するだけで同様の分析が可能です。テキスト前処理のトークナイザーも言語ごとに最適化されています。

**Q2: データの処理件数に上限はありますか？**

> A: ソフトウェアとしての上限はありません。バッチ処理（10件ずつ）でLLMに送信するため、大量データも効率的に処理できます。ただし、LLM APIの呼び出し回数に応じてコストが増加するため、事前に見積もり機能でコストを確認することを推奨しています。1万件規模のデータでも通常30分以内に処理が完了します。

**Q3: クラスタリングのアルゴリズムはどのように選べばいいですか？**

> A: 以下を目安にしてください。
> - **K-Means**: クラスタ数が事前にわかっている場合（例: 5つのカテゴリに分けたい）
> - **HDBSCAN**: クラスタ数が不明で、データに密度のばらつきがある場合（例: サポートチケットの自動分類）
> - **GMM**: テキストが複数のトピックにまたがる場合（例: アンケートの自由記述）

**Q4: 感情分析の精度はどの程度ですか？**

> A: LLM（大規模言語モデル）ベースのため、従来の辞書ベースやルールベースの手法と比較して高い精度を実現しています。特に日本語テキストの文脈理解に優れ、皮肉や婉曲的な表現も適切に判定できます。分析結果には信頼度スコアが付与されるため、信頼度の低い判定は人間が再確認することで精度を担保できます。

**Q5: 共起ネットワークのコミュニティ検出はどのアルゴリズムを使用していますか？**

> A: Louvainアルゴリズムを使用しています。モジュラリティ最大化により、ネットワーク内の密な結合パターンを自動検出します。Modularity値が0.3以上であれば、有意なコミュニティ構造が存在すると判断できます。

**Q6: AIエージェントのHITLモードの違いは何ですか？**

> A: 3つのモードがあります。
> - **Full Auto**: AIが完全に自律的に分析を進めます。スピード重視の場合に適しています。
> - **Semi-Auto**: 仮説生成の段階で人間の承認を求めます。分析の方向性をコントロールしたい場合に推奨です。
> - **Guided**: すべてのフェーズ（観測、仮説、探索、検証、統合）で承認を求めます。分析過程を詳細に把握したい場合に使用します。

**Q7: Groundingスコアとは何ですか？ ハルシネーションの対策は？**

> A: Groundingスコアは、AIが生成した主張がソースデータにどれだけ裏付けられているかを数値化したものです。AIの出力テキストとソースデータのコサイン類似度を計算し、0.0〜1.0のスコアとして表示します。0.8以上が推奨で、0.5未満の場合は元データを確認することを推奨しています。

### 導入・運用面の質問

**Q8: オンプレミス環境でも利用できますか？**

> A: はい。Docker環境があれば、オンプレミスで動作します。LLMについてもOllamaやvLLMを使用したローカルLLM実行に対応しているため、データを社外に出す必要はありません。AWS、Azure、GCPのクラウド環境にもTerraformで簡単にデプロイできます。

**Q9: データのセキュリティはどう確保されていますか？**

> A: 以下の対策を実施しています。
> - API通信はすべてHTTPSで暗号化
> - 個人情報は自動匿名化（PII Anonymizer機能搭載）
> - アクセス制御はJWTベースの認証
> - データの保存先はお客様指定の環境（クラウドストレージまたはオンプレミスDB）
> - SOC2 Type II準拠の運用体制

**Q10: 導入にはどの程度の期間が必要ですか？**

> A: Docker環境であれば、セットアップは30分程度で完了します。クラウド環境への本格導入は、環境構築・設定・テストを含めて約2〜4週間が目安です。データの準備やカスタマイズ要件によって変動します。

**Q11: 既存システムとの連携は可能ですか？**

> A: REST APIを提供しているため、既存のBIツール、CRM、データウェアハウスなどと柔軟に連携できます。Slack、Microsoft Teams、Salesforceとのインテグレーションも可能です。

**Q12: ライセンス形態はどうなっていますか？**

> A: Proprietary（プロプライエタリ）ライセンスです。利用ユーザー数に応じたサブスクリプション形式で提供しています。詳細は営業担当にお問い合わせください。

## トラブルシューティング

### 起動関連

**問題: `http://localhost:5173` にアクセスしても画面が表示されない**

| 確認項目 | 対処法 |
|---------|--------|
| Docker Composeが起動しているか | `docker compose ps` で全コンテナがrunningか確認 |
| フロントエンドコンテナのログ | `docker compose logs frontend` でエラーを確認 |
| ポート競合 | 他のアプリが5173ポートを使用していないか確認（`netstat -an | findstr 5173`） |
| ブラウザキャッシュ | Ctrl+Shift+Deleteでキャッシュをクリアし再読み込み |

**問題: `http://localhost:8000/api/health` がレスポンスを返さない**

| 確認項目 | 対処法 |
|---------|--------|
| バックエンドコンテナ | `docker compose logs backend` でエラーを確認 |
| `.env` ファイル | API キーが正しく設定されているか確認 |
| ポート競合 | 8000ポートが空いているか確認 |
| 再起動 | `docker compose restart backend` でバックエンドのみ再起動 |

### データインポート関連

**問題: CSVファイルをアップロードすると文字化けする**

| 原因 | 対処法 |
|------|--------|
| エンコーディング不一致 | CSVファイルをUTF-8（BOM付き）で保存し直す |
| Excel由来のCSV | ExcelからCSV保存時に「CSV UTF-8 (コンマ区切り)」を選択 |

**問題: 「対応していないファイル形式です」というエラーが出る**

| 原因 | 対処法 |
|------|--------|
| ファイル拡張子 | `.csv`, `.xlsx`, `.xls`, `.json`, `.tsv` のいずれかであることを確認 |
| MIMEタイプ | ファイルが破損していないか確認。再ダウンロードまたは再作成 |

**問題: カラムマッピングで「テキスト本文」を選択できない**

| 原因 | 対処法 |
|------|--------|
| 既に他のカラムに割り当て済み | 他のカラムの「テキスト本文」マッピングを「無視する」に変更してから再設定 |

### 分析実行関連

**問題: クラスタリング実行後、散布図に点が表示されない**

| 原因 | 対処法 |
|------|--------|
| データ件数不足 | 最低10件以上のテキストデータが必要 |
| テキスト内容が同一 | 同一テキストが多いと埋め込みが重なる。データ品質を確認 |
| ブラウザの描画問題 | ページをリロード（F5キー） |

**問題: 感情分析で「分析実行中...」のまま終わらない**

| 原因 | 対処法 |
|------|--------|
| LLM API応答遅延 | ネットワーク接続を確認。数分待っても完了しない場合はページをリロード |
| APIキー無効 | `.env` ファイルのAPIキーが有効か確認 |
| レート制限 | LLMプロバイダーのレート制限に達している場合、数分待ってから再試行 |

**問題: 共起ネットワークにノードが1つも表示されない**

| 原因 | 対処法 |
|------|--------|
| 最小出現頻度が高すぎる | スライダーを左に動かし、頻度閾値を下げる（例: 2〜3に設定） |
| データ件数不足 | 10件未満のデータでは共起パターンが検出されにくい |

### AIエージェント関連

**問題: 「分析開始」をクリックしても何も起きない**

| 原因 | 対処法 |
|------|--------|
| データセットIDが空 | データセットIDフィールドにインポート済みのIDを入力 |
| ボタンが無効化状態 | データセットIDが入力されていないとボタンは無効（グレーアウト） |

**問題: HITL承認パネルが表示されない**

| 原因 | 対処法 |
|------|--------|
| Full Autoモード選択 | Full Autoでは承認なしで最後まで進む。Semi-AutoまたはGuidedに変更 |
| 処理がまだ進行中 | フェーズ2（仮説）まで進むのを待つ |

### レポート生成関連

**問題: 「レポートを生成」ボタンがクリックできない**

| 原因 | 対処法 |
|------|--------|
| データセットIDが空 | データセットIDフィールドに値を入力 |

**問題: ダウンロードしたPPTXファイルが開けない**

| 原因 | 対処法 |
|------|--------|
| ファイル破損 | ネットワーク不安定により不完全なダウンロードの可能性。再度ダウンロード |
| Office未インストール | PowerPoint互換ソフトが必要。Google SlidesやLibreOffice Impressでも開ける |

### 画面表示関連

**問題: ダークモードに切り替えたい / 元に戻したい**

> サイドバー最下部の月アイコン（ダークモード切替）/ 太陽アイコン（ライトモード切替）をクリックします。

**問題: サイドバーが折りたたまれて操作しにくい**

> サイドバー最下部の矢印アイコン（`>` または `<` 折りたたむ）をクリックして展開します。

**問題: 画面が狭くてチャートが見切れる**

> ブラウザのズームを100%に設定し、F11キーで全画面表示にすることを推奨します。最低でも画面解像度1920x1080が必要です。

### エラーIDの活用

Sprint 18以降、全てのエラーレスポンスに `correlation_id`（エラーID）が付与されます。画面に表示されるエラーIDの先頭8文字を管理者に伝えることで、バックエンドログから該当エラーを特定できます。
